## Why Orator?

<CardGrid stagger>
	<Card title="Spec-first" icon="document">
		Write your OpenAPI spec, then generate the server code. Your spec is the source of truth.
	</Card>
	<Card title="Type-safe" icon="approve-check-circle">
		Every operation gets a trait method with proper request and response types — no stringly-typed nonsense.
	</Card>
	<Card title="Framework layers" icon="puzzle">
		Core generation is framework-agnostic. Axum support ships out of the box, with more frameworks to follow.
	</Card>
	<Card title="Just the boring bits" icon="setting">
		Orator generates the routing, parameter extraction, and serialisation. You write the interesting part — your business logic.
	</Card>
</CardGrid>
